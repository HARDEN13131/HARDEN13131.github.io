对知攻方能知防的进一步学习

前言

在上一次的学习之后，我对AI进行了进一步的问答，它提供了一个防守自动化的工具和思路
于是展开了今天的学习

🚀 第一步：准备战场（安装 Fail2ban）

安装完成后，我们可以用下面这个命令看看这位“新保安”有没有正式上岗:

sudo systemctl status fail2ban

如果看到绿色的 active (running)，说明它已经开始值班了！

🛠️ 第二步：配置小黑屋规则（Jail 配置）

Fail2ban 默认有一套规则，但为了配合我们昨天的红蓝对抗演练，我们要亲手给它写一个“只要 Kali 爆破超过 3 次就立刻封禁”的硬核规则。

  1. 在 Mint 终端里，使用纳米编辑器（nano）创建一个属于我们自己的配置文件：

    sudo nano /etc/fail2ban/jail.local

  2. 把下面这段配置直接复制粘贴进去：

    [sshd]
    enabled = true
    port = ssh
    logpath = %(sshd_log)s
    backend = %(sshd_backend)s

#最大失败次数：3次
maxretry = 3
#封禁时间：600秒（10分钟）
bantime = 600
#统计失败次数的窗口期：10分钟内失败3次就封
findtime = 600
3. 保存并退出
4. 让配置生效：重启一下 Fail2ban 服务：

sudo systemctl restart fail2ban

🥷 第三步：攻防测试（让 Kali 再次发起进攻！）

像昨天一样，再次对 Mint 发起 SSH 爆破或者直接故意输错密码登录

在三次错误后就会被permission denied

🛡️ 第四步：切回蓝队（Mint），查看战果

当你在 Kali 端被成功拦截后，切回你的 Linux Mint 终端，输入以下命令让 Fail2ban 保安汇报战绩

sudo fail2ban-client status sshd

看到输出里有一行叫 Banned IP list后面会清清楚楚地列着你 Kali 的 IP 地址

第五步：研究背后的原理（防止变成脚本小子doge）

fb2本质

Fail2ban 本质上是一个运行在 Linux 应用层的 Python 程序。它平时不做别的，就死死盯着你指定的日志文件（比如 /var/log/auth.log）

整个流程

盯梢 $\rightarrow$ 判刑 $\rightarrow$ 行刑

  1. 盯梢（应用层日志审计）

    当黑客在 Kali 端尝试 SSH 登录失败时，Linux 的 SSH 服务（sshd）就会在日志里写下一行：
    Jul 19 15:30:00 panda-vm sshd[2888]: Failed password for panda from 192.168.249.129 port 39929 ssh2

Fail2ban 内部有一套 正则表达式（Filter）

当它扫描到这行日志时，匹配到了 Failed password 和 IP 192.168.249.129，它的内部计数器就会给这个 IP 默默地记上一次
2. 判刑（触发 Jail 阈值）
当你在配置文件 jail.local 里设置了 maxretry = 3，意味着只要在 findtime（10分钟）内，同一个 IP 触发了 3 次这样的日志，Fail2ban 的控制核心就会发出警报
这时，Fail2ban 会根据你系统里的防火墙类型，自动生成一条只有计算机才能看懂的底层防火墙命令。
3. 行刑（深入 Linux 内核的 Netfilter）

在这里面发现了 只要访问目标端口是 22（SSH），并且它的源 IP 在上面那个名单簿里，通通拒绝（reject）tcp dport 22 ip saddr @addr-set-sshd reject
